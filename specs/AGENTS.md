# Specs

## Formato

Criar arquivo `.md` em `specs/`:

```markdown
# Nome da Feature - Especificação

## Visão Geral
Breve descrição do que resolve.

## Requisitos Funcionais
1. Requisito 1
2. Requisito 2

## Arquitetura e Tecnologias
### Bibliotecas
- Biblioteca X

### Estrutura de Componentes
```
ComponentePai
├── ComponenteFilho1
└── ComponenteFilho2
```

## Tasks
- [ ] Task 1
- [ ] Task 2
```

## Processo

1. Criar spec antes de implementar
2. Revisar spec antes de codar
3. Marcar tasks quando concluídas
